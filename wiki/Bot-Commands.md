## ⬇️ 下载控制

### 转发消息下载
**最简单的下载方式！**

直接转发任何包含媒体的消息给 Bot，自动下载。

**步骤**:
1. 找到要下载的消息
2. 长按 → 转发 → 选择你的 Bot
3. Bot 自动下载

**支持的媒体**:
- 📷 照片
- 🎥 视频
- 📄 文档
- 🎵 音频

**示例**:
```
[转发一个视频]

Bot: 🔄 检测到媒体文件，开始下载...
Bot: ✅ 下载完成

文件: video.mp4
大小: 150 MB
来源: 资源频道
```

---

### /browse
浏览频道历史消息列表。

**用法**:
```
/browse @channel [数量]
```

**参数**:
- `@channel` - 频道用户名或 ID
- `[数量]` - 可选，默认 20 条

**示例**:
```
/browse @my_channel 10
```

**返回**:
```
📺 我的频道
最近 10 条媒体消息:

消息 ID: 12345 | 06-16 14:30
🎥 视频 (200 MB)
📝 Python教程.mp4
💬 最新Python教程第一集

消息 ID: 12344 | 06-16 14:25
📄 文档 (5 MB)
📝 教程文档.pdf

...

💡 使用 /download 12345 下载指定消息
```

---

### /download
下载指定消息 ID 的媒体。

**用法**:
```
/download <消息ID> [频道]
```

**参数**:
- `<消息ID>` - 必需，消息的 ID
- `[频道]` - 可选，频道用户名

**示例**:
```
# 从已监听的频道下载
/download 12345

# 从指定频道下载
/download 12345 @my_channel
```

**返回**:
```
🔄 正在下载消息 12345...

✅ 下载完成！

文件: Python教程.mp4
```

---

### /history
批量下载历史消息。

**用法**:
```
/history @channel [数量]
```

**参数**:
- `@channel` - 频道用户名或 ID
- `[数量]` - 下载数量，0 = 全部

**示例**:
```
# 下载最近 100 条
/history @my_channel 100

# 下载全部历史
/history @my_channel 0
```

**返回**:
```
🔄 开始下载历史消息: 我的频道
限制: 100

请稍候...

✅ 历史下载完成！
成功下载 45 个文件
```

## ❓ 常见问题

**Q: 命令没有响应？**
A: 检查你的 User ID 是否在 `admin_ids` 中。

**Q: 转发下载不工作？**
A: 确保消息包含媒体文件。

**Q: /browse 显示空？**
A: 频道最近没有媒体消息。

**Q: /download 失败？**
A: 检查消息 ID 是否正确，或指定频道。
