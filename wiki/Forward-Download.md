# 转发下载教程

TgBackup 最简单、最直观的使用方式 - 转发即下载！

---

## 🎯 什么是转发下载？

看到任何有用的媒体文件，直接转发给 Bot，自动下载 - 就这么简单！

**无需**:
- ❌ 配置频道
- ❌ 手动命令
- ❌ 复杂操作

**只需**:
- ✅ 转发消息
- ✅ 等待下载
- ✅ 完成！

---

## 📱 使用步骤

### 步骤 1: 启动 Bot

确保你的 TgBackup Bot 正在运行：

```bash
docker ps
# 应该看到 tgbackup 容器
```

### 步骤 2: 找到你的 Bot

在 Telegram 中搜索你的 Bot 用户名（创建时设置的）

点击 "Start" 或发送 `/start`

### 步骤 3: 转发消息

1. 找到要下载的消息（包含照片、视频、文档或音频）
2. 长按消息
3. 选择 "转发"
4. 选择你的 Bot
5. 发送

### 步骤 4: 等待完成

Bot 会自动：
1. 检测媒体文件
2. 开始下载
3. 保存到本地
4. （可选）上传到 OneDrive
5. 通知你完成

## 📂 文件保存位置

### 本地存储

转发下载的文件会保存在：

```
downloads/
└── Manual/
    └── [来源名称]/
        └── 文件名
```

**示例**:
```
downloads/
└── Manual/
    ├── 学习群/
    │   └── Python教程.mp4
    ├── 资源频道/
    │   └── 文档.pdf
    └── 私聊/
        └── 照片.jpg
```

### OneDrive（如果启用）

同步到 OneDrive 的相同路径：

```
/TG_Media/Manual/[来源名称]/文件名
```

## ✨ 支持的媒体类型

### 📷 照片
- 格式: JPG, PNG, WEBP
- 大小: 按配置限制
- 质量: 原图

### 🎥 视频
- 格式: MP4, AVI, MKV, MOV, WMV
- 大小: 按配置限制（默认 2GB）
- 质量: 原视频

### 📄 文档
- 格式: PDF, DOCX, XLSX, TXT, ZIP, RAR...
- 大小: 按配置限制
- 完整: 原文件

### 🎵 音频
- 格式: MP3, M4A, OGG, FLAC
- 大小: 按配置限制
- 质量: 原音频

## 🎯 过滤规则

转发下载也会应用配置的过滤规则：

### 文件大小限制

```json
{
  "file_size_limit": 1073741824  // 1GB
}
```

如果转发的文件超过限制：

```
Bot: "⏭️ 文件已存在或不符合过滤条件"
```

### 扩展名黑名单

```json
{
  "extensions_blacklist": [".exe", ".bat"]
}
```

转发 .exe 文件会被跳过。

### 媒体类型限制

```json
{
  "media_types": ["video", "document"]
}
```

只接受视频和文档，照片会被跳过。

## ❓ 常见问题

### Q: 转发后没有反应？

**可能原因**:
1. Bot 没有启动
2. 你的 User ID 不在 `admin_ids` 中
3. 消息不包含媒体

**解决方法**:
```bash
# 检查 Bot 状态
docker logs tgbackup

# 检查配置
cat config/telegram_config.json
# 确认 admin_ids 包含你的 ID
```

---

### Q: Bot 说文件已存在？

**原因**: 文件已经下载过了（根据消息 ID 判断）

**如果确实想重新下载**:
```
/clear  # 清理下载记录
# 然后重新转发
```

---

### Q: 可以转发多个文件吗？

**可以！** 
- 快速连续转发多个文件
- Bot 会排队依次下载
- 完成后逐个通知

---

### Q: 转发的文件会自动上传 OneDrive 吗？

**取决于配置**:
```json
{
  "auto_upload": true  // 会自动上传
}
```

如果启用，转发下载的文件也会自动上传。

---

### Q: 如何知道来源？

Bot 会在下载完成消息中显示：

```
✅ 下载完成
文件: video.mp4
大小: 150 MB
来源: 资源频道  ← 这里
```

---

### Q: 可以转发私聊的文件吗？

**可以！** 只要是包含媒体的消息都能转发下载。

---

### Q: 转发相册怎么办？

**逐个转发**: Telegram 的相册是多个独立消息，需要逐个转发。

或者使用 `/browse` + `/download` 批量下载。

---

### Q: 下载失败怎么办？

**查看错误信息**:
```
Bot: "❌ 下载失败: [错误原因]"
```

**常见原因**:
- 文件太大
- 网络问题
- 磁盘空间不足
- 文件被删除

**解决方法**:
```
/logs 50  # 查看详细日志
/disk     # 检查磁盘空间
```
